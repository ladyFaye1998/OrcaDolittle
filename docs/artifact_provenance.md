# Zenodo and Git provenance

The large derived-artifact package for this submission is frozen on Zenodo. No new
Zenodo version is required for the current submission.

- Public concept DOI: [10.5281/zenodo.21030081](https://doi.org/10.5281/zenodo.21030081)
- Published package: `orcadolittle_derived_artifacts_clean_20260630T103554Z.zip`
- Package-internal manifest files: `PACKAGE_MANIFEST.json`, `PACKAGE_README.md`,
  and `SHA256SUMS.txt`
- Source commit recorded inside the published ZIP:
  `94f790f76d68fdd334769daa5eb1bd5fea2fc399`

The public Git repository is the live code and documentation surface. Commits after the
Zenodo package was sealed clarify DOI links, README/site wording, provenance notes, and
presentation details. They do not change the deposited large-artifact hashes, the numeric
metrics used for the submission claims, or the generated result figures.

For review, treat the two public objects as complementary:

| Object | Role |
|---|---|
| GitHub repository | Current code, README, documentation, small committed artifacts, scripts, tests, figures, and claim boundaries. |
| Zenodo concept DOI | Frozen large derived artifacts and package-level checksums for files that do not belong in Git. |

The concept DOI is intentionally cited everywhere instead of a one-off version DOI, so it
remains a stable public handle for the deposited artifact package. A new Zenodo upload
would only be needed for future work that changes large artifacts or reported results.
